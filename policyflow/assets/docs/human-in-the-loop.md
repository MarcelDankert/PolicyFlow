# Human In The Loop

Human approvals are part of the governance model, not an afterthought.

## Default Intent

- LOW risk: minimize interruption
- MEDIUM risk: escalate when rules or uncertainty require it
- HIGH risk: human approval required

## What Human Approval Should Cover

- protected business logic changes
- critical execution boundary changes
- database schema changes
- contract changes
- security and deployment pipeline changes
- architecture boundary changes

## Approval Evidence

Approval evidence should be explicit and reviewable:
- approver
- date
- scope approved
- restrictions
